# Инструкция по генерации оффлайн-тайлов

## Обзор

Этот документ описывает процесс генерации оффлайн-тайлов для карты Кольчугино с использованием GitHub Actions.

## Форматы тайлов

### MBTiles (рекомендуется)
- **Формат**: Единый SQLite файл (.mbtiles)
- **Преимущества**: 
  - Компактное хранение (сжатие внутри БД)
  - Быстрый доступ к тайлам
  - Удобно для распространения
- **Недостатки**: Требует библиотеку для чтения в браузере

### XYZ (для совместимости)
- **Формат**: Папки `{z}/{x}/{y}.png`
- **Преимущества**:
  - Простота использования
  - Совместимость с любыми картными библиотеками
- **Недостатки**: Много файлов, занимает больше места

## Настройка GitHub Actions

### 1. Включение GitHub Pages

1. Перейдите в настройки репозитория → **Pages**
2. Источник: **GitHub Actions**
3. Сохраните

### 2. Запуск генерации тайлов

#### Автоматический запуск по расписанию
Workflow настроен на запуск 1-го числа каждого месяца в 03:00 UTC.

#### Ручной запуск
1. Перейдите во вкладку **Actions** репозитория
2. Выберите workflow **"Generate Offline Map Tiles"**
3. Нажмите **"Run workflow"**
4. Укажите параметры:
   - **Минимальный зум**: 10-12 (рекомендуется 10)
   - **Максимальный зум**: 14-16 (рекомендуется 14)
   - **Название района**: Kolchugino

### 3. Параметры масштабирования

| Зум | Детализация | Количество тайлов | Размер (примерно) |
|-----|-------------|-------------------|-------------------|
| 10-12 | Область/город | ~100 | 5-10 МБ |
| 10-14 | Город/районы | ~1,000 | 50-100 МБ |
| 10-16 | Улицы/дома | ~10,000 | 500+ МБ |

**Рекомендация**: Используйте зум 10-14 для баланса детализации и размера.

## Кастомизация стилей

### Цветовая схема
В файле `tiles-config/style.json` можно изменить:

- **Фон за пределами района**: `#b8d4e8` (голубоватый)
- **Водоемы**: `#3498db` с паттерном волн
- **Лесные массивы**: `#27ae60` с паттерном деревьев
- **Парки**: `#2ecc71`
- **Дороги**: `#f39c12` (насыщенный оранжевый)
- **Здания**: `#e74c3c` (полупрозрачный красный)

### Добавление текстур
Для паттернов (волны, деревья) используются спрайты из OpenMapTiles.

## Административные границы

### Автоматическое получение
Workflow автоматически загружает границы Кольчугинского района через Overpass API.

### Ручное обновление границ
1. Скачайте GeoJSON с [Geofabrik](https://download.geofabrik.de/) или [Overpass Turbo](https://overpass-turbo.eu/)
2. Замените файл `data/boundaries/kolchugino_district.geojson`
3. Запустите генерацию тайлов

## Результаты генерации

После успешного выполнения workflow:

### Артефакты (доступны 30 дней)
- `kolchugino-offline.mbtiles` - MBTiles файл
- `xyz/` - папка с XYZ тайлами
- `preview.png` - превью карты

### Публикация на GitHub Pages
XYZ тайлы публикуются по адресу:
```
https://{username}.github.io/{repo}/offline-tiles/{z}/{x}/{y}.png
```

## Интеграция с плагином

### Настройка URL тайлов

В админке WordPress укажите:
- **URL оффлайн-тайлов**: `https://{username}.github.io/{repo}/offline-tiles/`
- **Мин. зум**: 10
- **Макс. зум**: 14

### Использование MBTiles в браузере

Для работы с MBTiles добавьте библиотеку:
```html
<script src="https://cdn.jsdelivr.net/npm/pmtiles@2.11.0/dist/index.min.js"></script>
```

Или конвертируйте MBTiles в PMTiles:
```bash
pmtiles convert kolchugino-offline.mbtiles kolchugino-offline.pmtiles
```

## Обновление тайлов

### При изменении данных
1. Внесите изменения в GeoJSON файлы
2. Запустите workflow вручную
3. Дождитесь публикации на GitHub Pages

### Плановое обновление
Workflow автоматически запускается 1-го числа каждого месяца.

## Устранение проблем

### Ошибка: "Failed to fetch from Overpass"
- Попробуйте запустить workflow повторно
- Используйте альтернативный источник (Geofabrik)

### Ошибка: "No tiles generated"
- Проверьте корректность границ в GeoJSON
- Увеличьте диапазон зумов

### Большой размер тайлов
- Уменьшите максимальный зум (например, с 16 до 14)
- Сократите область покрытия
