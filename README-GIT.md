# Синхронизация с GitHub

## Настройка

1. **Установка токена GitHub**
   ```bash
   export GITHUB_TOKEN=ваш_токен_доступа
   ```
   Токен должен иметь права на запись в репозиторий.

2. **Инициализация Git репозитория**
   ```bash
   chmod +x sync-to-github.sh
   ./sync-to-github.sh --init
   ```

3. **Добавление удаленного репозитория**
   ```bash
   ./sync-to-github.sh --add-remote
   ```

## Использование

### Основная синхронизация
```bash
# Синхронизация с сообщением коммита
./sync-to-github.sh "Обновление версии плагина до 1.0.63"

# Синхронизация с автоматическим сообщением
./sync-to-github.sh
```

### Проверка состояния
```bash
# Показать статус Git
./sync-to-github.sh --status

# Показать разницу
./sync-to-github.sh --diff

# Показать ветки
./sync-to-github.sh --branches

# Показать удаленные репозитории
./sync-to-github.sh --remotes
```

### Справка
```bash
# Показать помощь
./sync-to-github.sh --help
```

## Функции скрипта

- **Автоматическая инициализация** Git репозитория
- **Проверка токена** GitHub
- **Получение изменений** с сервера
- **Добавление всех изменений** в индекс
- **Создание коммита** с настраиваемым сообщением
- **Отправка на GitHub**
- **Обработка конфликтов** при pull
- **Цветной вывод** для удобства

## Требования

- Git
- Токен GitHub с правами на запись
- Доступ к удаленному репозиторию

## Безопасность

- Токен хранится в переменной окружения
- Никогда не передается в командной строке
- Используется HTTPS для безопасной передачи данных
