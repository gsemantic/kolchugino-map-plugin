# Структура проекта Kolchugino Map Plugin

Этот документ описывает архитектуру и структуру файлов плагина карты для Кольчугино.

## 📂 Корневая директория

| Файл / Папка | Описание |
| :--- | :--- |
| `kolchugino-map.php` | **Главный файл плагина**. Содержит заголовок плагина, точку входа и подключение основных классов. |
| `readme.txt` | Стандартный файл readme для репозиториев WordPress плагинов (используется для каталога wordpress.org). |
| `README.md` | Основная документация проекта для разработчиков и пользователей на GitHub. |
| `includes/` | Папка с PHP-классами, реализующими логику плагина. |
| `assets/` | Статические ресурсы: CSS, JavaScript, шрифты, изображения. |
| `templates/` | PHP-шаблоны для вывода фронтенд-части плагина. |
| `data/` | Геоданные (GeoJSON) с точками интереса (POI) и другими объектами карты. |
| `.gitignore` | Список файлов и папок, игнорируемых системой контроля версий Git. |

## 📂 Детальная структура

### 1. Логика (includes/)
Здесь находится объектно-ориентированный код плагина.

*   `class-post-type.php` — Регистрация пользовательского типа записей для объектов карты.
*   `class-taxonomy.php` — Регистрация таксономий (категорий, тегов) для объектов.
*   `class-metabox.php` — Добавление мета-боксов для ввода координат и дополнительных данных в админке.
*   `class-settings.php` — Страница настроек плагина в админ-панели WordPress.
*   `class-ajax-handler.php` — Обработчики AJAX-запросов для динамической подгрузки данных на карту.
*   `class-export.php` — Логика экспорта данных в различные форматы (GeoJSON, KML и др.).
*   `class-shortcode.php` — Регистрация шорткода `[kolchugino_map]` для вставки карты на страницы.
*   `class-logger.php` — Утилита для логирования событий и ошибок (для отладки).
*   `index.php` | Пустые файлы для защиты от просмотра содержимого директории через браузер.

### 2. Ресурсы (assets/)
Клиентская часть плагина.

*   `css/map.css` — Стили для отображения карты и маркеров.
*   `js/map.js` — Основной скрипт инициализации карты (OpenLayers/Leaflet).
*   `js/map-admin.js` — Скрипты для административной панели (выбор точек на карте при редактировании).
*   `js/map-offline.js` — Логика работы карты в офлайн-режиме (если предусмотрено).
*   `vendor/` — Сторонние библиотеки (например, OpenLayers: `ol.js`, `ol.css`).
*   `.htaccess` — Правила доступа для веб-сервера Apache.

### 3. Данные (data/)
Хранилище географических данных.

*   `kolchugino_poi.geojson` — Основной файл с точками интереса (POI) города Кольчугино.
*   *(Тестовые файлы исключены из репозитория)*

### 4. Шаблоны (templates/)
Файлы для рендеринга HTML.

*   `map-container.php` (предположительно) — Шаблон обертки карты.
*   `index.php` — Защита директории.

## 🔄 Поток данных

1. **Админка**: Пользователь добавляет объекты через пост-тип -> Данные сохраняются в БД WordPress.
2. **Экспорт**: Класс `Export` может выгружать данные из БД в `data/*.geojson`.
3. **Фронтенд**: Шорткод вызывает `Shortcode` -> Подключает `assets/js/map.js` -> Скрипт загружает GeoJSON -> Отрисовывает карту через библиотеку в `vendor`.

## 🛠 Установка и разработка

1. Скопируйте папку плагина в `wp-content/plugins/`.
2. Активируйте плагин в админке WordPress.
3. Для разработки клонируйте репозиторий и установите зависимости (если есть composer/npm).
