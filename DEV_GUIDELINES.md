# Руководство по разработке (DEV_GUIDELINES)

## 1. Общие правила
- Все изменения в коде вносятся только в директории `/workspace`.
- Перед началом работы обязательно ознакомьтесь с текущей структурой проекта и состоянием файлов.
- После внесения изменений всегда проверяйте работоспособность функционала.

## 2. Работа с Git и публикация изменений
- **Критически важно:** Для публикации изменений на GitHub используйте **только** скрипт `./sync-to-github.sh`.
- Ручное выполнение команд `git add`, `git commit`, `git push` запрещено во избежание рассинхронизации.
- Использование скрипта:
  ```bash
  ./sync-to-github.sh "Описание внесенных изменений"
  ```
- Скрипт автоматически выполняет:
  - `git pull` (получение актуальной версии)
  - `git add` (добавление измененных файлов)
  - `git commit` (фиксация изменений с сообщением)
  - `git push` (отправка на удаленный репозиторий)
- Успешное выполнение скрипта активирует кнопку публикации в интерфейсе.

## 3. Стиль кода и архитектура
- **Стандарт JavaScript:** Используется **ES6+** (стрелочные функции, `const`/`let`, шаблонные строки, модули и т.д.) как основной стандарт разработки.
- **Совместимость:** При использовании синтаксиса ES6+ учитывайте требования к поддержке браузеров. Если проект должен работать в старых браузерах без процесса транспиляции (Babel), используйте только те возможности ES6+, которые поддерживаются целевыми браузерами, или придерживайтесь ES5 для критических участков.
- Придерживайтесь существующего стиля кода в проекте.
- Избегайте добавления лишних зависимостей без согласования.
- Комментарии к коду должны быть краткими и информативными.

## 4. Специфические настройки проекта (Карта)
- Контролы карты (`attribution`, `rotate`, `zoom`) настраиваются в файле `assets/js/map.js`.
- По умолчанию скрыты элементы управления: `attribution: false`, `rotate: false`.

## 5. Процесс работы
1. **Старт:** Прочтение этого файла и анализ текущей задачи.
2. **Разработка:** Внесение изменений в файлы проекта.
3. **Финал:** 
   - Проверка работоспособности.
   - Обязательный запуск `./sync-to-github.sh` для сохранения результатов.
   - Обновление документации при необходимости.
