# Исправление проблемы с отображением слоёв карты

## Что было исправлено

### Проблема:
1. **При локальных тайлах** - виден только слой с точками, подложки нет
2. **При онлайн тайлах (OSM)** - только подложка, точек нет (возможно они внизу)

### Причина:
Неправильный порядок слоёв при инициализации карты OpenLayers. Векторный слой с маркерами либо перекрывался тайлами, либо был под ними.

### Решение:
1. **При инициализации карты** - теперь явно указываем порядок слоёв в массиве `layers: [tileLayer, vectorLayer]`
2. **Добавлен `zIndex: 1000`** для векторного слоя, чтобы гарантировать его отображение поверх тайлов
3. **При переключении режимов** - используем `insertAt(0)` для тайлового слоя, чтобы он всегда был внизу
4. **Добавлен `zIndex: 0`** для тайловых слоёв при переключении

## Как протестировать

1. **Тест с локальными тайлами:**
   - Откройте страницу с картой
   - Должны быть видны И локальные тайлы, И точки на карте
   - Откройте консоль браузера (F12) и проверьте логи:
     - "Отрисовка маркеров: X объектов"
     - "Добавлено X фич в векторный источник"
     - "Количество слоев на карте: 2"

2. **Тест с онлайн тайлами:**
   - Нажмите кнопку "Оффлайн режим" (иконка download)
   - Должны переключиться на тайлы OSM
   - Точки должны остаться видимыми поверх тайлов
   - Проверьте консоль на наличие ошибок

3. **Обратно на локальные тайлы:**
   - Снова нажмите кнопку режима
   - Должны вернуться локальные тайлы с точками

## Отладка

Если проблема сохраняется, откройте консоль браузера и проверьте:

```javascript
// Какие слои есть на карте
map.getLayers().forEach((layer, i) => {
    console.log(`Слой ${i}:`, layer);
});

// Сколько объектов загружено

```
